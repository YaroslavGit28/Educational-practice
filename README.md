# Документация проекта (оглавление)

Эта папка содержит подробную документацию по проекту интернет‑магазина канцтоваров на Django.

## Навигация
- `01_overview_and_run.md` — назначение проекта, сценарии, как запустить (Windows/PowerShell)
- `02_architecture.md` — архитектура, приложения, потоки данных
- `03_roles_and_permissions.md` — роли, доступы, кто что может
- `04_models.md` — модели, поля, связи (ER‑описание)
- `05_routes_web.md` — web‑маршруты (страницы сайта)
- `06_routes_api.md` — API (DRF): эндпоинты, аутентификация, примеры запросов
- `07_ui_templates.md` — шаблоны, layout, страницы, где что лежит
- `08_ajax_frontend.md` — AJAX‑логика: какие формы перехватываются, какие JSON‑ответы ожидаются, что обновляется в DOM
- `09_admin_and_panels.md` — Django Admin + кастомная админ‑панель + панели менеджера/склада
- `10_seed_and_test_data.md` — тестовые данные, команда `seed_data`, картинки товаров
- `11_submission_checklist.md` — чек‑лист “готово к сдаче”
- `12_troubleshooting.md` — частые ошибки и как чинить

## Важно (про БД)
Проект **не использует `BD.sql` автоматически**. Django использует БД, заданную в `magazin/settings.py` (по умолчанию `db.sqlite3`).
`BD.sql` — это SQL‑скрипт/схема (справочник/дамп), который применяется только вручную.
