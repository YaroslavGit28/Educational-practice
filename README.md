# Интернет-магазин канцтоваров

Полнофункциональное веб-приложение интернет-магазина канцелярских товаров, разработанное на фреймворке Django в рамках учебной практики.

## Описание

Проект представляет собой современный интернет-магазин с полным циклом работы: от витрины товаров до обработки заказов сотрудниками. Реализованы ролевая модель, корзина покупок, оформление заказов, административные панели и REST API.

### Основные возможности

- **Каталог товаров** - иерархические категории, карточки товаров, изображения
- **Корзина покупок** - работает для гостей и авторизованных пользователей
- **Оформление заказов** - с регистрацией и без (гостевой заказ)
- **Ролевая система** - покупатели, менеджеры, склад, администраторы
- **Обработка заказов** - смена статусов, история изменений
- **Админ-панель** - кастомный интерфейс для управления контентом
- **REST API** - полный API с токеновой аутентификацией
- **Swagger документация** - интерактивная документация API
- **Отчеты** - статистика продаж, экспорт в Excel

## Технологический стек

- **Backend:** Python 3.11+, Django 4.2
- **API:** Django REST Framework, drf-yasg (Swagger)
- **База данных:** SQLite (по умолчанию) / PostgreSQL
- **Frontend:** HTML5, CSS3, JavaScript, Bootstrap 5
- **Дополнительно:** Pillow (изображения), openpyxl (Excel), python-dotenv

## Быстрый старт

### Требования

- Python 3.11 или выше
- pip (менеджер пакетов Python)
- Git (для клонирования репозитория)

### Установка

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/YaroslavGit28/Educational-practice.git
cd internet_magazin
```

2. **Создайте виртуальное окружение (рекомендуется):**
```bash
# Windows
python -m venv venv
venv\Scripts\activate
```

3. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

4. **Настройте переменные окружения:**
```bash
# Скопируйте пример файла .env
copy .env.example .env  # Windows
cp .env.example .env    # Linux/macOS

# Отредактируйте .env файл при необходимости
# (для разработки можно оставить значения по умолчанию)
```

5. **Выполните миграции базы данных:**
```bash
python manage.py migrate
```

6. **Создайте суперпользователя (администратора):**
```bash
python manage.py createsuperuser
```

7. **Соберите статические файлы:**
```bash
python manage.py collectstatic --noinput
```

8. **Запустите сервер разработки:**
```bash
python manage.py runserver
```

Приложение будет доступно по адресу: **http://127.0.0.1:8000/**

## 👥 Роли пользователей

### Покупатель (customer)
- Просмотр каталога товаров
- Добавление товаров в корзину
- Оформление заказов
- Просмотр истории заказов
- Редактирование профиля

### Менеджер (manager)
- Все возможности покупателя
- Подтверждение заказов
- Отмена заказов
- Просмотр всех заказов
- Управление через админ-панель

### Склад (warehouse)
- Все возможности покупателя
- Сборка заказов (статус "assembling")
- Отметка об отправке (статус "shipped")
- Управление остатками товаров

### Администратор (admin)
- Полный доступ ко всем функциям
- Управление пользователями
- Управление каталогом товаров
- Настройка системы
- Просмотр отчетов

## 🔧 Конфигурация

### Переменные окружения (.env)

| Переменная | Описание | Пример |
|------------|----------|--------|
| `SECRET_KEY` | Ключ безопасности Django | `django-insecure-change-me` |
| `DEBUG` | Режим отладки | `True` / `False` |
| `DB_ENGINE` | Движок БД | `sqlite` / `postgres` |
| `DB_NAME` | Имя базы данных | `db.sqlite3` |
| `DB_USER` | Пользователь БД (для PostgreSQL) | `postgres` |
| `DB_PASSWORD` | Пароль БД (для PostgreSQL) | `password` |
| `DB_HOST` | Хост БД (для PostgreSQL) | `localhost` |
| `DB_PORT` | Порт БД (для PostgreSQL) | `5432` |

### Использование с PostgreSQL

1. Установите PostgreSQL и создайте базу данных
2. В файле `.env` установите:
```env
DB_ENGINE=postgres
DB_NAME=internet_magazin
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
```
3. Установите адаптер PostgreSQL:
```bash
pip install psycopg2-binary
```
4. Выполните миграции:
```bash
python manage.py migrate
```

## 📊 Модели данных

### Основные сущности

- **User** - пользователь (email, роль, профиль)
- **Category** - категория товаров (иерархическая структура)
- **Product** - товар (название, цена, остаток, изображение)

- **Order** - заказ (статус, сумма, доставка)

- **OrderStatus** - статус заказа
- **OrderStatusHistory** - история изменений статуса

## 🔌 REST API

Проект предоставляет полный REST API для всех основных операций.

### Аутентификация
API использует токены для аутентификации. Получите токен через эндпоинт `/api/token-auth/`.

### Примеры запросов

**Получение списка товаров:**
```bash
curl -X GET http://localhost:8000/api/products/ \
  -H "Authorization: Token YOUR_TOKEN"
```

**Добавление товара в корзину:**
```bash
curl -X POST http://localhost:8000/api/cart/add/ \
  -H "Authorization: Token YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"product_id": 1, "quantity": 2}'
```

**Создание заказа:**
```bash
curl -X POST http://localhost:8000/api/orders/create/ \
  -H "Authorization: Token YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"delivery_address": "ул. Примерная, д. 1"}'
```

### Swagger документация
Интерактивная документация API доступна по адресу: **http://localhost:8000/api/docs/**
