## 2. Выбранный вариант развертывания

**Выбран вариант C:** локальный demo-стенд через Docker (`docker-compose.prod.yml`).

Такой вариант выбран потому, что проект — серверное Django-приложение с SQLite, миграциями, media/static и health endpoint. Для учебной практики нет внешнего VPS, поэтому production-compose на локальной машине является рабочим demo-стендом.

Дополнительно подготовлен release-архив `release/project_release.zip` как переносимый демонстрационный вариант.

---

## 3. Где открыть результат

Файл `public_url.txt` содержит адрес demo-стенда и краткое описание способа развертывания.

**Адрес:** http://127.0.0.1:8000/

---

## 4. Команды развертывания

| Действие | Команда | Результат |
|----------|---------|-----------|
| Проверка перед развертыванием | `scripts\deploy.bat` | Сборка и запуск `docker-compose.prod.yml` |
| Проверка доступности | `scripts\check_deploy.bat` | Health endpoint и главная страница |
| Повторный запуск | `scripts\restart.bat` | Перезапуск контейнеров |
| Сборка переносимого архива | `scripts\build_release.bat` | `release\project_release.zip` |
| Просмотр логов | `scripts\logs.bat` | Логи контейнера |

Основной CI workflow находится в `.github\workflows\ci.yml`.

---

## 5. Переменные окружения

Реальные секреты в репозиторий не попадают. Для этапа 3 добавлены безопасные примеры demo/production-конфигурации.

| Файл | Что настраивается | Секреты |
|------|-------------------|---------|
| `.env.production.example` | APP_ENV, DEBUG=false, SECRET_KEY, ALLOWED_HOSTS, DEPLOY_TARGET, PUBLIC_URL | Не содержит реальных значений |
| `.env.demo.example` | APP_ENV=demo, DEBUG=false, PUBLIC_URL, DATABASE_URL | Не содержит реальных значений |

## 6. Проверка доступности

| Проверка | Что проверяет | Ожидаемый результат |
|----------|---------------|---------------------|
| `scripts\check_deploy.bat` | Health и главная | HTTP 200 |
| `scripts\deploy.bat` | Запуск production-compose | Контейнер в статусе Up |
| `scripts\build_release.bat` | Сборка demo-архива | `release\project_release.zip` |
| Открытие http://127.0.0.1:8000/ | Доступность вне IDE | Каталог товаров |
| Основной сценарий | Вход, корзина, заказ | Заказ создаётся |
| `scripts\restart.bat` | Повторный запуск | Сервис снова доступен |
