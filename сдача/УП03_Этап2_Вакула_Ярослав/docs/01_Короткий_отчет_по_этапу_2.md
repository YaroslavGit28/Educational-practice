## 2. Как запустить проект локально

Проект является серверным Django-приложением с базой данных SQLite. Для Windows подготовлены BAT-скрипты и Makefile.

| Действие | Команда | Результат |
|----------|---------|-----------|
| Установка зависимостей | `scripts\setup.bat` | Создаёт `.venv`, устанавливает пакеты из `requirements.txt` |
| Создание настроек | `copy .env.example .env` | Появляется файл окружения |
| Запуск приложения | `scripts\run.bat` | Сервер на http://127.0.0.1:8000/ |
| Проверка проекта | `scripts\check.bat` | `manage.py check`, тесты, compileall |
| Форматирование | `scripts\format.bat` | black/ruff (при установке dev-зависимостей) |

---

## 3. Docker и обоснование контейнеризации

Docker для данного проекта **используется**, потому что проект относится к категории, где контейнеризация оправдана по методичке:

| Критерий | Состояние проекта | Вывод |
|----------|-------------------|-------|
| Backend | Django 4.2.7 | Docker упрощает повторяемый запуск |
| База данных | SQLite + volume | Compose фиксирует окружение |
| Несколько сервисов | web + volumes | `docker-compose.yml` нужен |
| Зависимости | Python, Pillow, openpyxl | Образ исключает ручную настройку |
| Запуск | `runserver` / gunicorn | Единая команда `docker-up.bat` |

Подготовлены: `Dockerfile`, `docker-compose.yml`, `.dockerignore`, `scripts\docker-up.bat`, `scripts\docker-down.bat`, `scripts\logs.bat`.
